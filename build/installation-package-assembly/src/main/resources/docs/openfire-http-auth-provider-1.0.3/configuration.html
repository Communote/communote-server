Configuration

Before you can use the HTTPRequestAuthProvider you have to edit the file http-auth.properties to configure the authentication provider. After editing the file you have to restart Openfire.

http-auth.properties

The property file should be copied to the OPENFIRE_INSTALL_FOLDER/conf.

It has the following properties:

Property Example Value Description
url http://{HOST}:{PORT}/{CLIENT}/authenticate/xmpp.do The url which will be used. The protocol could be http oder https.
HOST,CLIENT, LOGIN and PASSWORD are placeholders replaced by the provider.
fallback.provider.auth.className org.jivesoftware.openfire.ldap.LdapAuthProvider Additional authenticator, which should be used before the HTTP request is used.
param.login username Post parameter name for the login
param.password password Post parameter name for the password
host localhost Default host
port 8080 Default port
client myClient Default client
pattern.login [a-zA-Z.]+(@.*)? Regular expression to match logins for HTTP request.
pattern.login.fallback [a-zA-Z]+(@.*)? Regular expression to match logins for fallback method
pattern.host [a-zA-Z0-9_\-\.]+\.([a-zA-Z0-9_\-]+)\.[0-9]+ Regular expression to extract the host from the login.
pattern.port [a-zA-Z0-9_\-\.]+\.([0-9]+) Regular expression to extract the port from the login.
pattern.client [a-zA-Z0-9_\-\.]+\.([a-zA-Z0-9_]+)\.[a-zA-Z0-9_]+\.[0-9]+ Regular expression to extract the client from the login.
allowed_user[i].user
allowed_user[i].password		 # admin/123456
allowed_user[0].user=admin
allowed_user[0].password=7c4a8d09ca3762af61e59520943dc26494f8941b
allowed_user[1].user=otherUser@host
allowed_user[1].password=7c4a8d09ca3762af61e59520943dc26494f8941b		 Internal login list with username and password as SHA-1 hash. "i" has to start at 0 and must be incremented for each entry pair.
user_host[i].user
user_host[i].host		 user_host[0].user=user@com29
user_host[0].host=com234		 Predefined host for specific users.

Alternative location of http-auth.properties

Generally the http-auth.properties file should be found automatically. If this doesn't work you can specify its path manually in the openfire.xml. Simple add the following:

<jive>
  ...
  <!-- Path path to the HttpAuthenticationProvider properties file. -->
  <authProvider>
    <!-- The full path to the configuration file -->
    <confFile>/PATH_TO_CONF_FILE/http-auth.properties</confFile>
  </authProvider>
</jive>
