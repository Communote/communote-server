About

The HTTPRequestAuthProvider is an implementation for authenticating users via HTTP requests. It uses the Apache HTTPClient, which is already shipped with Openfire, for the request. So http and https requests are possible by default.

It can extract several parameters from the users jid and replace placeholders within the authentication url with that.

If the request returns the HTTP status 200 (OK) the user is authenticated, else not.

The authenticator also provides a simple fallback mechanism and an internal list for authenticating users.

Order of authentication

  1. Internal list
  2. Fallback authentication (if defined)
  3. HTTP requesdt authentication
